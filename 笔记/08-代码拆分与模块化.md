# 08 - 代码拆分与模块化重构

## 问题

所有代码都挤在 BattleScene.ts 一个文件里（1769 行），
技能、HUD、敌人AI、BOSS、玩家移动……全混在一起。
想改一个技能的参数要在文件里翻半天，加新关卡更是噩梦。

## 解决方案：模块化拆分

把"独立的功能"拆到各自的文件，BattleScene 只负责"串起来"。

### 拆分后的目录结构

```
game/src/
  types/
    EnemyTypes.ts        ← 枚举 + 接口（纯数据定义，无运行时代码）
  hud/
    PlayerHud.ts         ← 玩家HUD（血条 + 蓝条 + 技能CD图标）
    BossHud.ts           ← BOSS血条（屏幕顶部大血条）
  skills/
    FlameDash.ts         ← 技能1：烈焰闪
    GiantSword.ts        ← 技能2：巨剑术
  entities/
    PlayerGraphics.ts    ← 玩家绘制（已有）
  scenes/
    BattleScene.ts       ← 战斗场景（1369行，还在继续瘦身）
```

## 拆分原则

### 1. 类型定义先行（EnemyTypes.ts）
枚举和接口是最底层的依赖，其他文件都引用它。
它只有类型没有代码，不会产生循环依赖。

### 2. HUD 自包含（PlayerHud / BossHud）
HUD 类接收 scene 引用，自己创建和管理所有 UI 元素。
BattleScene 只需：
```typescript
this.hud = new PlayerHud(this, skill1Cost, skill2Cost);
this.hud.updateHp(hp, maxHp);      // 每帧调用
this.hud.updateSkillCd(...);
```

### 3. 技能类 = 参数 + 状态 + 执行逻辑（FlameDash / GiantSword）
每个技能是独立类：
- **参数**作为 readonly 字段（改数字就在文件顶部）
- **CD 计时**作为公共属性（BattleScene 读取用于 HUD 显示）
- **execute(ctx)** 方法接收上下文，返回 boolean（是否成功释放）

BattleScene 的技能处理简化为：
```typescript
if (key pressed && mp >= skill.mpCost) {
  if (skill.execute(ctx)) this.playerMp -= skill.mpCost;
}
```

### 4. SkillContext 接口（解耦关键）
技能需要访问场景、玩家、敌人，但不能直接依赖 BattleScene。
通过接口传递依赖：
```typescript
interface SkillContext {
  scene: Phaser.Scene;
  player: Container;
  facingRight: boolean;
  enemies: Group;
  damageEnemy: (enemy, damage, knockback) => void;  // 回调
  setDash: (dashing, endTime) => void;               // 回调
}
```

回调函数让技能能"通知"BattleScene 做事（比如进入冲刺状态），
而不需要知道 BattleScene 内部怎么实现。

## 拆分顺序很重要

从"依赖最少"的开始拆：
1. EnemyTypes（纯类型，零依赖）
2. PlayerHud / BossHud（只依赖 Enemy 类型）
3. Skills（依赖 EnemyTypes + Scene）
4. 后续：Enemies、Player（依赖更多，耦合更深）

## 重构后的好处

- 改技能参数 → 打开 skills/FlameDash.ts，顶部就是数字
- 改 HUD 样式 → 打开 hud/PlayerHud.ts，只有 UI 代码
- 加新技能 → 新建文件，实现 execute()，BattleScene 加一行调用
- 编译验证 → 每拆一个模块就 `npx tsc --noEmit`，确保零错误
